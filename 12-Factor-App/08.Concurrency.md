# 8. Concurrency

## Vertical Scaling
We will run a docker container on an instance to handle several amount of users. As users increase we need more containers and there will be a limit to which we can increase the container in huge application on an instance.
There will be a downtime for doing it. It can be done for application where downtime does not make impact on production.

But where 24/7 availability of the server is mandate it fails. There we need different approach for doing it.

## Horizontal Scaling
Rather than scaling containers, we can scale small size instances for the same app container and keep load balancers to route the traffic to the designated instances. To work it as expected we need our app to be an independent stateless app.
