# 5. Build, Release & Run

12 Factor app uses the strict separation between Build, Release & Run stages.

## Build Stage
The stage where mostly the code is build & stored as a file format, and then converted into a executable binary format to run the application. Converting the code from a text format to a executable format is the Build phase. One of the tool to do it is a docker. We simply run docker build command to make it happen.

## Release Stage
The build image generated from the Build stage along with the .env file stored separately on the environment becomes a release object in Release stage. Every release should have its own release id for easy recognition.

## Run Stage
This is the stage where the release object is actually in running state in respective environments.

These are the 3 stages which helps us ensuring apps are running with same codebase in the respective environments in a consistent fashion.

So any minor change such as a TYPO in the codebase also have to get observed without miss. In this case to avoid creating new release for a minor change, we can rollback to previous version as we have those details saved in prior release and after making the change we can rerelease the new version.
