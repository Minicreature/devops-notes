# 6. Processes

12 factor app are stateless & share nothing. Sticky sessions are violation of 12 factor app and should never be used or relied upon.

Redis like application helps to avoid the violation & store session related data to other location than the host of application. In this case, all the instances running for same application will store the data at a unified location from where they can gather the session information without the hassle. If an instance is suddenly having some issue, your session info is safe for another instance to work on it.
