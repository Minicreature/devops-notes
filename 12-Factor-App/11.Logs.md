# 11. Logs

Log files are used to manage visit, login, error, logout, etc. journey details.
Traditionally it was stored on the same server as host. But in the world of containers it is easy to close the container. So in this case log data can be lost after taking the container down.

To prevent it from happening:

A 12 factor app never concerns itself with routing or storage of its output stream.
Store the logs in centralised location in structured format.

12 factor app must write its logs to its STDOUT or to a local file as a .json in a structured format. Then be used by an agent to centralised location to consolidate & analysis.

There are tools like ELK stack, fluentd, splunk, etc. to do the centralised logging solution.
